Step-by-step setup guide

How to publish your website with GitHub + Netlify

Written for complete beginners. No prior experience needed. You will have your site live at yazanawwad.com by the end of this guide — free, fast, and fully in your control.

What we'll do
  1. Install the tools (GitHub Desktop)
  2. Create your GitHub account
  3. Upload your website files to GitHub
  4. Create your Netlify account and connect GitHub
  5. Connect your domain (yazanawwad.com)
  6. Update your site any time
  7. Add a new article
  8. Set up your contact form (Formspree)
1
Install GitHub Desktop

GitHub Desktop is a free app that lets you manage your website files without any complicated commands. Think of it like a "save + upload" button for your website.

Download it here: desktop.github.com

→
Go to the link above and click "Download for Windows" (or Mac if you're on a Mac)
→
Open the downloaded file and install it like any normal program
→
Don't open it yet — we'll set it up in Step 2
2
Create your GitHub account

GitHub is a website where your code files are stored safely online. It's free. Think of it like Google Drive — but specifically for website files.

Go here: github.com/signup

→
Enter your email address, create a password, and choose a username.
Tip: Use something professional like yazanawwad or yazan-awwad — this will be visible in URLs
→
Verify your email address (GitHub sends you a confirmation email)
→
You're in. Now open GitHub Desktop and click "Sign in to GitHub.com" — log in with your new account
✓ Good to know GitHub is free forever for personal websites and public projects. You don't need to pay anything.
3
Upload your website files to GitHub

First, make sure your website files are organised in one folder on your computer. They should look like this:

yazan-site/ ├── index.html ← your homepage ├── articles.html ← your articles list page ├── css/ │ ├── style.css │ └── article.css ├── js/ │ └── main.js └── articles/ ├── norway-growth.html ├── customer-service-systems.html └── mis-design-thinking.html
→
In GitHub Desktop, click "Create a New Repository on your hard drive..."
→
Name it yazan-awwad-website (no spaces). Choose where to save it on your computer. Click Create Repository
→
Open that folder on your computer (click "Show in Explorer" or "Show in Finder" in GitHub Desktop). Copy all your website files into it — all the HTML files, the css folder, the js folder, and the articles folder
→
Go back to GitHub Desktop. You'll see all the files listed with green "+" icons — this means they're new files ready to be saved
→
At the bottom-left, you'll see a box. Type a short message like First version of my site in the Summary field. Then click Commit to main
→
Now click "Publish repository" at the top. Make sure "Keep this code private" is unchecked (it needs to be public for Netlify to read it). Click Publish Repository
✓ What just happened? Your files are now stored on GitHub's servers. Go to github.com and log in — you'll see your repository listed there.
4
Create your Netlify account and connect GitHub

Netlify is the service that takes your files from GitHub and makes them available as a live website. It's free, and it updates your site automatically every time you change a file.

Go here: app.netlify.com/signup

→
Click "Sign up with GitHub" — this connects your Netlify account to your GitHub account automatically
→
Once you're in, click the button "Add new site" → then "Import an existing project"
→
Click "Deploy with GitHub". Netlify will ask permission to access your GitHub — click Authorize
→
You'll see your repositories listed. Click yazan-awwad-website
→
On the next screen, leave everything as-is. Just scroll down and click "Deploy yazan-awwad-website"
→
Wait about 30 seconds. Netlify will give you a temporary URL like random-name-123.netlify.app — click it to see your live site!
✓ Your site is now live on the internet Anyone with that Netlify URL can see it. In the next step, we'll connect your real domain (yazanawwad.com).
5
Connect your domain (yazanawwad.com)

This step points your existing domain to Netlify so that when someone goes to yazanawwad.com, they see your new site. You'll need to log into wherever you bought your domain (this is usually GoDaddy, Namecheap, Google Domains, or similar).

→
In Netlify, go to your site dashboard → click "Domain management" → then "Add a domain"
→
Type yazanawwad.com and click Verify → then Add domain
→
Netlify will show you some DNS records to add. These look like:

Type: A · Name: @ · Value: 75.2.60.5
Type: CNAME · Name: www · Value: your-site.netlify.app
→
Log into your domain registrar (GoDaddy, Namecheap, etc.). Find the DNS settings for yazanawwad.com. Add the records exactly as Netlify shows them
→
Wait up to 24 hours (usually much faster — often under 1 hour). Your site will appear at yazanawwad.com with a free HTTPS certificate (the padlock)
⚠ Important If you're currently using WordPress at yazanawwad.com, the moment you change the DNS records, your WordPress site will go offline and your new site will appear. Make sure you're ready to switch before doing this step.
✓ Not sure which registrar you use? Go to lookup.icann.org, type yazanawwad.com, and look for the "Registrar" field — it tells you exactly who manages your domain.
6
Update your site any time

This is the real power of this setup. Updating your site is now a simple 3-step process that takes about 60 seconds:

1
Edit your file. Open the file you want to change in any text editor (Notepad, VS Code, anything). Make your changes. Save the file.
2
Open GitHub Desktop. You'll see your changed files listed. Write a short note about what you changed (e.g. Updated about section). Click Commit to main.
3
Click "Push origin". That's it. Netlify detects the change automatically and updates your live site in about 30 seconds.
✓ Recommend: VS Code for editing Download VS Code — it's free, it highlights code in colour, and it makes editing HTML much easier than Notepad. When you open your site folder in VS Code, all files are shown in a sidebar and you can click to edit any of them.
7
Add a new article

Every time you write something new, here is the exact process:

1
Go to your articles/ folder. Find any existing article file (e.g. norway-growth.html). Copy it and rename the copy to something like my-new-article.html
2
Open your new file. Change:
• The <title> tag at the top to your article title
• The post-tag text (the category)
• The article-title heading
• The date and read time in article-byline
• All the paragraphs inside article-body
• The "Next article →" link at the bottom to point to another article
3
Open articles.html. Copy one of the existing <a class="article-card"> blocks and paste a new one. Update the href, title, excerpt, and date to match your new article
4
Open index.html. In the "Writing" section, add a new post-preview-card link pointing to your new article
5
Save all files → GitHub Desktop → Commit → Push. Done. Your article is live.
✓ Shortcut You can also send me your article text and I'll generate the complete HTML file for you. Then you just drop it into the folder and push. One minute of work on your end.
8
Set up your contact form (Formspree)

The contact form on your site needs a service to actually send emails. We use Formspree — it's free for up to 50 submissions per month and takes 3 minutes to set up.

Go here: formspree.io/register

→
Sign up with your Gmail (yazanawwad61@gmail.com) — this is where form submissions will be sent
→
Once inside, click "+ New Form". Name it Contact Form. Click Create Form
→
Formspree gives you a form endpoint URL that looks like: https://formspree.io/f/xyzabc12. Copy it.
→
Open index.html. Find this line:
action="https://formspree.io/f/YOUR_FORM_ID"
Replace YOUR_FORM_ID with your actual ID from Formspree
→
Save → GitHub Desktop → Commit → Push. Test it by submitting the form on your live site — you'll receive the email in your Gmail within seconds
